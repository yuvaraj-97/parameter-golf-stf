Speed Reality

Freezing should make later layers faster only when frozen tokens are actually skipped in compute, not merely marked frozen in bookkeeping.

Does freezing make future layers faster?

Only if the implementation actually stops sending frozen tokens through later layer compute. If the code keeps full tensors and just masks, blends, or reuses frozen states, the model may be logically frozen but not much faster.

Old logs show whole-step average time, not per-layer skip. New STF telemetry can now expose the key reality check directly: what percentage of token-layer work was logically frozen versus actually skipped.
